# P03 数据构造 — perf-data-builder

## 解决的问题

造数据慢、参数化 CSV 手工拼、数据量不足导致压测中途数据耗尽、缓存命中率虚高导致 TPS 失真。

## 触发词

- 构造压测数据 / 压测数据准备 / test data / data builder
- perf data / 造数据

## 输入

- 数据库表结构 DDL / 接口文档 / 数据量需求

## 输出

| 输出类型 | 适用场景 |
|----------|---------|
| INSERT SQL | DDL 模式：按表依赖顺序批量造数据 |
| CSV 参数化文件 + 请求体模板 | 接口模式：直接喂给 JMeter |
| 数据量计算 | 自动计算需造数据量 |

---

## 数据量计算公式

```
所需数据量 = 并发数 × TPS/线程 × 持续时间(秒) × 去重系数
```

| 场景类型 | 去重系数 | 说明 |
|----------|---------|------|
| 读接口（支持重复数据） | 1.0 | 少量数据即可 |
| 写接口（需唯一数据） | 1.5~2.0 | 每条数据唯一 |
| 分页查询 | 1.2~1.5 | 需覆盖多页 |
| 用户维度隔离 | 并发数 × 1.5 | 每用户独占数据 |

## DDL 模式（推荐）

步骤：
1. 输入表结构 DDL（CREATE TABLE 语句）
2. 分析表间外键/依赖关系
3. 按依赖顺序生成 INSERT 语句
4. 加入业务合理的数据分布（比如不同状态的数据各占一定比例）

INSERT 语句质量要求：
- 支持批量插入（单条 INSERT 多行）
- 按依赖拓扑排序（先主表后子表）
- 区分读数据（可重复用）和写数据（需唯一）

## 接口模式

输入接口文档后输出：

1. **CSV 参数化文件**（JMeter CSV Data Set Config 直接可用）
   ```
   userId,productId,quantity,couponId
   u001,p001,1,c001
   u002,p002,2,c002
   ```

2. **请求体模板**（对应各场景）
   ```json
   {
     "userId": "${userId}",
     "productId": "${productId}",
     "quantity": ${quantity},
     "couponId": "${couponId}"
   }
   ```

## 数据分布原则

| 数据类型 | 分布比例建议 |
|----------|-------------|
| 正常状态数据 | 70% |
| 边界状态数据（空/零/满） | 15% |
| 异常状态数据 | 10% |
| 极端数据（超长/超大） | 5% |

## 跨平台 Prompt 模板

```
你是一个性能测试数据构造工程师。根据用户提供的输入（DDL/接口文档/数据量需求），生成压测数据。

数据量计算公式: 并发数 × TPS/线程 × 持续时间(秒) × 去重系数
去重系数：读接口 1.0，写接口 1.5-2.0，分页查询 1.2-1.5，用户维度隔离 并发数×1.5

处理模式：

模式1: DDL 模式（用户提供表结构 DDL）
  步骤:
  1. 分析表间外键依赖关系
  2. 按依赖顺序生成批量 INSERT 语句
  3. 数据分布原则: 正常 70% + 边界 15% + 异常 10% + 极端 5%
  4. 写数据需唯一标识，读数据可重复使用

模式2: 接口模式（用户提供接口文档）
  步骤:
  1. 生成 CSV 参数化文件（JMeter CSV Data Set Config 可用格式）
  2. 生成对应请求体模板（${变量名} 格式）
  3. 确保 CSV 行数 ≥ 所需数据量

输出要求：
- CSV 文件: 首行为列名，逗号分隔，无多余空格
- INSERT SQL: 一条 INSERT 含多行 VALUES，用分号分隔
- 附带数据总量说明和预计覆盖的时长
```
