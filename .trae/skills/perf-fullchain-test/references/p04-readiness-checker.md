# P04 就绪检查 — perf-readiness-checker

## 解决的问题

环境不稳定、监控没配、脚本跑不通、数据未就绪，导致压测中途中断或结果失真。

## 触发词

- 压测就绪检查 / readiness check / 压测前检查
- perf readiness / check before test

## 输入

- 环境信息 / 脚本文件 / 监控配置 / 数据准备状态

## 输出

- 4 大维度 34 项检查清单及结论（Pass / Fail / Warning）
- 不通过项的排查思路和常见根因

---

## 检查清单

### 维度一：环境（10 项）

| 编号 | 检查项 | 检查方法 |
|------|--------|---------|
| ENV-01 | 目标服务状态正常 | curl health endpoint / telnet port |
| ENV-02 | 数据库连接正常 | 通过 JMeter JDBC 或直连验证 |
| ENV-03 | Redis 连通正常 | redis-cli ping |
| ENV-04 | MQ 连通正常 | 生产/消费验证 |
| ENV-05 | 压测机到目标网络延迟/丢包 | ping / mtr |
| ENV-06 | 测试环境数据库连接池大小 ≥ 目标并发 | 查询配置 |
| ENV-07 | Tomcat/Netty 线程池 ≥ 目标并发 | 查询配置 |
| ENV-08 | 中间件版本与生产一致 | 版本核对 |
| ENV-09 | DNS 解析正常 | nslookup |
| ENV-10 | 防火墙/安全组未限制压测流量 | iptables / 安全组规则检查 |

### 维度二：脚本（9 项）

| 编号 | 检查项 | 说明 |
|------|--------|------|
| SCRIPT-01 | 线程组配置正确 | 并发数、Ramp-up、循环次数 |
| SCRIPT-02 | HTTP 请求路径/方法/参数正确 | 与实际接口一致 |
| SCRIPT-03 | 参数化文件路径正确 | CSV Data Set Config 引用正确 |
| SCRIPT-04 | 断言配置完整 | 响应码 + 业务码 + 响应时间断言 |
| SCRIPT-05 | 关联提取器配置正确 | JSON Extractor / Regex Extractor |
| SCRIPT-06 | 监听器配置（查看结果树非必需） | Aggregate Report / Backend Listener |
| SCRIPT-07 | 测试片段/模块控制器引用正确 | 模块化脚本时 |
| SCRIPT-08 | Cookie/Token 管理器配置 | 认证通过 |
| SCRIPT-09 | 写接口幂等性验证 | 重复请求不产生脏数据 |

### 维度三：监控（8 项）

| 编号 | 检查项 | 说明 |
|------|--------|------|
| MON-01 | JMeter 聚合报告已配置 | 基础指标收集 |
| MON-02 | 服务端 CPU 监控 | top / nmon / Prometheus |
| MON-03 | 服务端内存监控 | free / /proc/meminfo |
| MON-04 | 服务端磁盘 IO 监控 | iostat / dstat |
| MON-05 | 服务端网络监控 | sar -n DEV / iftop |
| MON-06 | 数据库监控（慢查询/连接数） | slow_query_log / show processlist |
| MON-07 | GC 日志已开启 | -Xloggc |
| MON-08 | 监控大盘或告警已配置 | Grafana / Zabbix |

### 维度四：数据（7 项）

| 编号 | 检查项 | 说明 |
|------|--------|------|
| DATA-01 | 数据量是否满足压测时长 | >= 预估消耗量 |
| DATA-02 | 数据分布是否合理（覆盖多种状态） | 正常/边界/异常 |
| DATA-03 | 参数化数据是否去重 | 避免重复导致断言失败 |
| DATA-04 | 写操作数据清理方案已准备 | Truncate / 软删除还原 |
| DATA-05 | 缓存已预热（如需要） | 防止冷缓存 RT 虚高 |
| DATA-06 | 无脏数据残留 | 上次压测数据已清理 |
| DATA-07 | 共享数据竞争条件已评估 | 如库存扣减类 |

## 不通过项排查指引

### 环境类常见问题

| 问题 | 排查步骤 |
|------|---------|
| 服务不可达 | 1. telnet IP PORT → 2. curl health → 3. 检查负载均衡/网关路由 → 4. 查看服务日志 |
| 数据库连接失败 | 1. 连接串/账号密码 → 2. 白名单 → 3. 最大连接数 → 4. DB 状态 |
| 连接池满 | 1. show processlist 看活跃连接 → 2. 检查 `max_connections` → 3. 释放空闲连接 |

### 脚本类常见问题

| 问题 | 排查步骤 |
|------|---------|
| 401/403 认证失败 | 1. 检查 Token/Cookie 提取器 → 2. 检查 Header 管理器 → 3. 检查登录前置脚本 |
| 参数化数据读取失败 | 1. CSV 文件路径 → 2. 编码 UTF-8 → 3. 变量名引用正确 → 4. 文件末尾无空行 |
| 断言误判 | 1. 查看结果树查看实际响应 → 2. 调整断言表达式 → 3. 区分响应码断言和业务码断言 |

## 跨平台 Prompt 模板

```
你是一个性能测试就绪检查官。根据用户提供的环境/脚本/监控/数据信息，执行 4 维度 34 项检查，输出检查报告。

检查维度：

1. 环境(10项): 服务状态、DB连接、Redis、MQ、网络延迟、连接池大小、线程池、中间件版本、DNS、防火墙
2. 脚本(9项): 线程组配置、HTTP参数、参数化路径、断言(响应码+业务码+响应时间)、关联提取、监听器、模块引用、认证、写幂等性
3. 监控(8项): 聚合报告、CPU、内存、磁盘IO、网络IO、DB慢查询、GC日志、监控大盘
4. 数据(7项): 数据量≥消耗量、数据分布合理、参数化去重、清理方案、缓存预热、脏数据清理、竞争条件

输出格式：

## 就绪检查报告
### 环境 | 脚本 | 监控 | 数据
每项标注: ✅ Pass / ❌ Fail / ⚠️ Warning

## 不通过项详情
- 项目: [编号]
- 问题描述
- 排查思路（具体命令/步骤）
- 预期修复方法

## 总体结论
- ✅ 全部通过 → 可启动压测
- ⚠️ 有 Warning → 记录风险，可启动
- ❌ 有 Fail → 必须修复后再启动
```
