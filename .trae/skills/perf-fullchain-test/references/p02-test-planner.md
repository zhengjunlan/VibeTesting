# P02 测试计划 — perf-test-planner

## 解决的问题

场景参数凭感觉填、压测机资源没估算、数据量没评估、预热遗漏、基线对比口径不统一。

## 触发词

- 压测计划 / 性能测试计划 / test plan
- perf test plan / plan performance test

## 输入

- P01 需求澄清文档 / 直接提供的压测参数

## 输出

1. 场景参数推导（线程数、Ramp-up、持续时间）
2. 场景组合推荐（四段式递进）
3. 压测机资源估算（JMeter 节点数）
4. 数据量风险评估（数据耗尽预警）
5. 数据预热计划
6. 基线对比计划

---

## 场景参数推导

| 参数 | 推导公式 |
|------|---------|
| 线程数 | 目标并发数 |
| Ramp-up | 并发数 / 5（秒），最少 60 秒 |
| 持续时间 | 基线 5 分钟 / 容量每阶梯 5-10 分 / 稳定性 12 小时 |
| 循环次数 | 永久（通过持续时间控制） |

## 四段式场景组合

```
阶段1: 单接口基线测试
  目的: 验证每个接口独立性能
  执行: 每个接口单独压测，获取基线数据

阶段2: 混合场景测试
  目的: 模拟业务真实比例流量
  执行: 按业务比例分配线程（如 下单:查询:支付 = 3:5:2）

阶段3: 极限容量测试
  目的: 找到系统崩溃点/性能拐点
  执行: 阶梯加压（50→100→200→400→800），每阶梯持续 5 分钟

阶段4: 稳定性测试
  目的: 验证长时间运行的性能衰减
  执行: 70%~80% 目标并发下持续 12 小时
```

## 压测机资源估算

| 场景 | 建议规格 |
|------|---------|
| 并发数 < 500 | 单台 4C8G JMeter 足够 |
| 并发数 500~2000 | 2 台 4C8G 或 1 台 8C16G |
| 并发数 2000~5000 | 3~4 台 8C16G（分布式压测） |
| 并发数 > 5000 | 5 台以上 + 网络带宽检查 |

> 单台 JMeter 不建议超过 2000 并发线程，否则压测机本身 CPU 会成为瓶颈。

## 数据量风险评估

计算公式：
```
所需数据量 = 并发数 × TPS/线程 × 持续时间(秒) × 去重系数

示例: 500 并发 × 1 TPS/线程 × 600 秒(10分钟) × 1.2 去重系数
     = 360,000 条数据
```

预警规则：
- 可获取数据量 < 所需数据量 80% → **高危**，需提前造数据
- 可获取数据量 80%~120% → **中危**，需分批使用或增加去重
- 可获取数据量 > 120% → **正常**

## 数据预热计划

判断是否需要预热：
| 条件 | 需要预热 |
|------|---------|
| 涉及数据库查询 | ✅ 是 |
| 涉及 Redis 缓存 | ✅ 是 |
| 涉及本地缓存 | ✅ 是 |
| 纯静态资源/Cache 命中率 > 95% | ❌ 否 |

预热方式：正式压测前以 20%~30% 并发运行 1~2 分钟，等待 RT 稳定进入 warmed-up 状态后开始记录数据。

## 基线对比计划

| 对比项 | 说明 |
|--------|------|
| 对比版本 | 本次压测版本号 |
| 基线版本 | 上次压测版本号 / 初始版本 |
| 对比指标 | TPS、平均 RT、P50、P95、P99、错误率 |
| 对比阈值 | TPS 下降 > 10% 或 RT 增加 > 15% 视为退化 |

## 跨平台 Prompt 模板

```
你是一个性能测试计划设计师。根据用户提供的需求文档或参数，输出标准化的性能测试计划。

输出内容：

1. 场景参数表
   - 线程数、Ramp-up（并发数/5，最低60秒）、持续时间
   - 基线测试 5 分钟，容量规划每阶梯 5-10 分钟，稳定性 12 小时

2. 四段式场景组合
   - 阶段1: 单接口基线测试 —— 每个接口单独获取基线
   - 阶段2: 混合场景测试 —— 按业务比例（如 3:5:2）
   - 阶段3: 极限容量测试 —— 阶梯加压（50→100→200→400→800）
   - 阶段4: 稳定性测试 —— 70%~80% 目标并发持续 12 小时

3. 压测机资源估算
   - <500 并发: 1台 4C8G
   - 500~2000: 2台 4C8G 或 1台 8C16G
   - 2000~5000: 3~4台 8C16G
   - >5000: 5台以上

4. 数据量风险评估
   - 公式: 并发数 × TPS/线程 × 持续时间(秒) × 1.2
   - 预警: <80% 高危需造数据，80~120% 中危需分批，>120% 正常

5. 数据预热判断
   - 涉及 DB 查询/Redis/本地缓存 → 需要预热
   - 预热方式: 20%~30% 并发预跑 1~2 分钟

6. 基线对比计划
   - 对比版本、基线版本、对比指标(TPS/RT/P50/P95/P99/错误率)
   - 退化阈值: TPS↓>10% 或 RT↑>15%
```
